# 🏀 [CAS] Baller 球员评分机制说明 (Rating Protocol)

> **机密等级**：公开 (Classified Assets)
> **版本**：v1.1 (优化版)

---

## 1. 核心维度定义 (Core Metrics)
系统从以下五个维度对球员进行“生物学建模”：

* **投篮 (Shooting)**：中远距离投射能力。
* **防守 (Defense)**：外线贴防与篮下护框意识。
* **物理 (Physical)**：基于身高、体重的对抗强度。
* **控球 (Dribbling)**：持球稳定性与摆脱能力。
* **耐力 (Endurance)**：全场奔跑能力与出勤韧性。

---

## 2. 综合评分计算 (Overall Calculation)

目前采用 **全员共识平均算法**，计算逻辑封装在数据库视图 `v_leaderboard` 中：

### 🧮 计算公式
1.  **单项分** = 该球员所有评价记录的算术平均值。
2.  **综合分 (Overall)** = (投篮Avg + 防守Avg + 物理Avg + 控球Avg + 续航Avg) / 5。

### ⚖️ 算法规则
* **实时性**：每新增一条评分，视图会自动重新计算。
* **四舍五入**：最终 Overall 采取四舍五入（如 79.6 → 80）。
* **权重**：当前权重比例为 **1:1:1:1:1**（全维度平权）。

系统根据平均分自动在前端渲染“身份勋章”：

| 标签           | 触发门槛             | 视觉呈现           |
| :------------- | :------------------- | :----------------- |

---

## 4. 排名机制 (Ranking)

* **首要排序**：按 `Overall` 综合分从高到低排列。
* **次要排序**：若分值相同，则按 `vote_count`（被评价次数）排序。
* **活跃度**：高赞“垃圾话”评论会增加球员在社区的活跃权重。

---

## 5. 数据安全与修正
* **反作弊**：每个用户对同一名球员只能评价一次。
* **匿名性**：打分过程匿名，保护评价者（特工）的身份。
